C++ std::bit_cast와 reinterpret_cast — 언제 어떤 것을 써야 하는가

C++ 타입 퍼닝에서 bit_cast와 reinterpret_cast의 올바른 사용법

C++MLOpstype punningstrict aliasingsystem optimization
원본 보기 →
핵심 포인트
· std::bit_cast는 비트 패턴 복사에 안전한 구문 편의 기능이지만 const 제거 등 오용 가능
· reinterpret_cast는 엄격한 앨리어싱 규칙 위반으로 미정의 동작 유발
· memcpy를 통한 타입 퍼닝이 C++ 표준에서 정의된 안전한 방법
적용 아이디어
💡 ML 모델의 바이너리 직렬화/역직렬화 시 std::bit_cast 대신 memcpy를 사용해 엄격한 앨리어싱 규칙 위반 방지
💡 고성능 데이터 파이프라인에서 uint8_t* 포인터를 float*로 변환 시 reinterpret_cast 대신 std::bit_cast(src) 대신 memcpy로 안전하게 변환
퀴즈
Q1. 타입 퍼닝 시 엄격한 앨리어싱 규칙을 위반하지 않는 방법은?
A) reinterpret_cast(&i)
B) union { uint32_t i; float f; }
C) std::memcpy(&f, &i, sizeof(f))
D) const_cast(reinterpret_cast(src))
정답 보기
✓ C) std::memcpy(&f, &i, sizeof(f))
memcpy는 바이트 단위 복사로 새로운 객체를 생성하므로 엄격한 앨리어싱 규칙을 위반하지 않습니다. reinterpret_cast와 union은 미정의 동작일 수 있습니다.
Q2. std::bit_cast(src)와 reinterpret_cast(src)의 주요 차이점은?
A) bit_cast는 const 제거 가능
B) bit_cast는 비트 복사만 수행
C) bit_cast는 객체 수명 관리
D) bit_cast는 포인터 변환 불가
정답 보기
✓ B) bit_cast는 비트 복사만 수행
std::bit_cast는 비트 패턴을 복사하는 기능으로 포인터 변환은 지원하지 않습니다. reinterpret_cast는 포인터/참조자 간 변환에 사용되며 엄격한 앨리어싱 규칙을 무시할 수 있습니다.
